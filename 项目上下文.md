## 核心设计（2026-06-23 更新）

### parse_file 返回 3 元组

```python
by_type, epochs, errors = parse_file("log.nmea")
```

- **by_type**: `{"GPGGA": [GGA, ...], "GNGGA": [GGA, ...], ...}` — 按 talker+type 分组，多星座不并桶
- **epochs**: `{"083000.000000": [GGA, RMC, GSA, ...], ...}` — 按 UTC 时间戳（含微秒）将同一**精确瞬间**的语句聚成历元；5Hz/10Hz 高频接收机每秒多次定位各自独立
- **errors**: 解析错误列表

### 校验分两级

1. **语句级**（_check_gga / _check_rmc / ...）：逐条检查字段合法性
2. **历元级**（_check_epoch_duplicates）：同一精确历元内 GGA/RMC/VTG 应唯一，GSA/GSV 允许多条。历元 key 带小数秒，高频定位不会误报

### 行号

每条语句记录 `line_number`（parse_file 时设置），校验报告用文件行号定位问题。

## 已覆盖的校验规则

- 校验和验证（XOR）
- 定位质量等级识别（RTK固定/浮点/单点/差分/无效）
- 经纬度范围检查（±90°/±180°）
- 卫星数合理性
- HDOP/PDOP/VDOP 精度因子
- 差分龄期检查
- SNR 信号强度
- 卫星仰角
- **历元内唯一性**（同一精确瞬间出现多条 GGA/RMC/VTG → error；历元 key 含小数秒，5Hz/10Hz 高频不会误报）

### 🥇 项目2：GNSS 设备模拟器（Mock Device）

**要解决的问题**：嵌入式设备测试的痛点——没设备的时候没法测。写一个脚本模拟 RTK 接收机往外发数据（TCP），让上层软件测试不依赖真实硬件。

**技术点**：
- 核心模拟器：起 TCP server，按 1Hz/5Hz 频率持续发 NMEA 数据
- 轨迹生成：直线、圆周、随机走动
- 故障注入：丢包、延迟、噪声、信号丢失（fix_quality 从 4→1→0）
- 多星座支持：同时发 GP + BD + GN 语句

**你已有的基础**：NMEA 格式你完全懂了，轨迹生成跟你的 GIS 空间分析经验对上。

---

### 🥈 项目3：坐标转换 API 测试框架

**要解决的问题**：中海达手簿软件的核心功能之一就是坐标转换（WGS84→CGCS2000→当地平面坐标），转换准不准直接影响测量成果。写一个专门测坐标转换服务的测试套件。

**技术点**：
- 封装常见坐标系变换（WGS84↔CGCS2000↔百度/高德）
- 往返一致性测试：A→B→A 误差不应超阈值
- 已知控制点对比：用国家大地控制点的标准坐标做期望值
- 边界点位：赤道、极地、换带边界、跨带

**你已有的基础**：ArcGIS 上坐标系投影你经常用，对七参数/四参数转换不陌生。
